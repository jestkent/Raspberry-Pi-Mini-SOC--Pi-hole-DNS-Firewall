What changed today

Added Unbound as a local recursive DNS resolver on the Pi and configured Pi-hole to forward queries to 127.0.0.1#5335. This reduces reliance on public DNS providers and improves privacy and control.

Client (Windows 11, iPad) ↓ Pi-hole (filter + logging) ↓ Unbound (127.0.0.1:5335, recursive resolver) ↓ Root DNS servers

Key commands used

Install Unbound

sudo apt update sudo apt install -y unbound

Create config directory

sudo mkdir -p /etc/unbound/unbound.conf.d

Create Unbound config file

sudo nano /etc/unbound/unbound.conf.d/pi-hole.conf

Root hints

sudo mkdir -p /var/lib/unbound sudo wget -O /var/lib/unbound/root.hints https://www.internic.net/domain/named.cache

Enable and start

sudo systemctl restart unbound sudo systemctl enable unbound sudo systemctl status unbound

Direct Unbound test

dig google.com @127.0.0.1 -p 5335

Validation evidence

From real-time Pi-hole logs, Pi-hole forwarded DNS queries to Unbound on localhost:

forwarded settings-win.data.microsoft.com to 127.0.0.1#5335 reply settings-prod-scus-2.southcentralus.cloudapp.azure.com is 52.xxx.xxx.xxx

Real-time monitoring command

sudo tail -f /var/log/pihole/pihole.log
